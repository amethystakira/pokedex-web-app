# Pokedex Web App

## Project Overview

This project is a **Pokedex web application** that lets users explore Pokemon data in a clean, interactive interface. The goal is to build a responsive app where users can browse Pokemon, search by name or number, sort results, and view more detailed information for each Pokemon.

This project is designed to be realistic for the course timeline while still leaving room for later milestones such as improved filtering, better UI polish, and expanded detail views.

## Purpose

The purpose of this project is to create a user-friendly web application that helps users quickly discover and compare Pokemon information. It also demonstrates how to work with a public API, dynamically render data in the browser, and build features like search, sorting, and filtering.

## Public API Used

This project uses the **[PokeAPI](https://pokeapi.co/)**.

Main endpoints used/planned:

- `https://pokeapi.co/api/v2/pokemon`
- `https://pokeapi.co/api/v2/pokemon/{id}`
- `https://pokeapi.co/api/v2/pokemon-species/{id}`

## Current Features

The current project structure already includes:

- A homepage for listing Pokemon
- Search input and sorting options in the interface
- A detail page structure for individual Pokemon
- Static assets and styling files

## Setup and Run

1. Clone or download the project folder.
2. Open the project in a code editor such as VS Code.
3. Run the project using a local server.

Simple options:

- Use the VS Code Live Server extension
- Or open `index.html` in the browser
